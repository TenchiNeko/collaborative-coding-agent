## CODING PATTERNS (ALWAYS FOLLOW)

### Python Basics
- NO TODO comments — write real, working code
- NO placeholder implementations
- Import ALL modules you use — especially `from datetime import datetime`
- Use `str(uuid.uuid4())[:8]` for IDs — always strings, never uuid.int
- `json.dump(data, f, default=str)` when data may contain datetime objects

### SQLite
- Always assign SQL to a variable, then pass to execute():
  ```python
  query = (
      "CREATE TABLE IF NOT EXISTS projects ("
      "  id INTEGER PRIMARY KEY AUTOINCREMENT,"
      "  name TEXT NOT NULL,"
      "  description TEXT"
      ")"
  )
  cursor.execute(query)
  ```
- Use parenthesized string concatenation for multi-line SQL (shown above)
- NEVER use backslash line continuation in strings
- Use parameterized queries: `cursor.execute("SELECT * FROM t WHERE id = ?", (id,))`

### Flask APIs
- Use `app.test_client()` in tests, NEVER call route functions directly
- Return proper status codes: 201 for creation, 204 for deletion, 400 for bad input, 404 for missing
- Validate input before database operations

### Dataclasses
- Always use `@dataclass` decorator syntax
- NEVER write `dataclass(ClassName):`
- Non-default fields MUST come before default fields:
  ```python
  @dataclass
  class Project:
      name: str                    # no default — MUST be first
      description: str             # no default — MUST be first
      id: str = field(default_factory=lambda: str(uuid4())[:8])
      created_at: datetime = field(default_factory=datetime.now)
  ```
- NEVER use mutable defaults like `id: str = str(uuid4())[:8]` — this creates ONE id shared by all instances. Use `field(default_factory=...)` instead.

### CLI Applications
- `main(argv=None, **deps) -> int:` with `parser.parse_args(argv)`
- Dependencies MUST be injectable parameters with defaults
- Provide a separate `build_parser()` helper
